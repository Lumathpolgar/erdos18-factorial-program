# Nova Startup Checklist

Each Nova must complete this checklist before beginning new theorem work.

## Repository and branch

- [ ] Read `README.md`.
- [ ] Read `docs/PROBLEM_STATEMENT.md`.
- [ ] Read `docs/CURRENT_STATUS.md`.
- [ ] Read `docs/COMMON_NOTATION.md`.
- [ ] Read `docs/SHARED_MATHEMATICAL_CONTRACT.md`.
- [ ] Read `docs/VERIFICATION_AND_EVIDENCE_STANDARD.md`.
- [ ] Read the assigned track charter.
- [ ] Confirm the assigned branch.
- [ ] Do not work directly on `main`.

## Historical reconstruction

- [ ] Read `archive/TRACK_A_RECORD.md`.
- [ ] Read `archive/TRACK_B_RECORD.md`.
- [ ] Read `archive/PHASE12K_12P_RECORD.md`.
- [ ] Identify which historical failures apply to the assigned route.
- [ ] Do not import an archived claim without checking its current status.

## Mathematical baseline

- [ ] Restate `h(N)`, `lambda_N(x)`, and `H_N(X)` exactly.
- [ ] Restate the primary factorial target.
- [ ] Restate the current sufficient half-range theorem.
- [ ] Distinguish proved, conditional, finite, heuristic, and disproved results.
- [ ] List every external theorem expected to be used.

## Track initialization

- [ ] Create `STATUS.md` from the status template.
- [ ] Create `THEOREMS.md` or the track-specific registry.
- [ ] Create `OPEN_REQUIREMENTS.md`.
- [ ] Record initial conjectures with status `OPEN` or `HEURISTIC`.
- [ ] Define verification commands.
- [ ] Commit the baseline before experimentation.

## First review checkpoint

Before promoting any theorem, confirm:

- [ ] exact hypotheses and conclusion;
- [ ] valid asymptotic range;
- [ ] divisor legality;
- [ ] numerical distinctness;
- [ ] term count;
- [ ] target uniformity;
- [ ] boundary cases;
- [ ] dependency registration;
- [ ] independent review request.
